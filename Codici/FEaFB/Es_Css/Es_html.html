Come funziona CSS: demo completa (con commenti)

CSS = regole + selettori + cascata

Spiegazione (in paragrafi)

CSS serve a descrivere lo stile di una pagina HTML: colori, dimensioni, spazi, allineamenti e animazioni. In pratica l’HTML dice “cos’è” un elemento (titolo, paragrafo, pulsante), mentre il CSS dice “come appare”.

Il CSS funziona tramite regole. Ogni regola ha un selettore (a chi si applica) e un blocco di proprietà (cosa cambia). Esempio mentale: “tutti i paragrafi hanno un certo colore e una certa distanza tra le righe”.

La parola “Cascading” indica che possono esistere più regole che puntano allo stesso elemento, e il browser decide quale vince usando specificità e ordine: in genere regole più specifiche o dichiarate dopo possono sovrascrivere quelle precedenti.

Il layout moderno usa spesso Flexbox (per allineare elementi in una riga/colonna) e Grid (per creare griglie a colonne/righe). In questa pagina, la sezione principale usa una griglia a due colonne e diventa a una colonna con una regola @media su schermi piccoli.

Un concetto fondamentale è il Box Model: ogni elemento ha contenuto, padding, bordo e margine. Se capisci il box model, capisci quasi sempre “perché” un elemento è troppo grande, troppo distante o non si allinea come ti aspetti.

Inoltre il CSS ha pseudo-classi come :hover (quando passi col mouse), :active (quando clicchi) e :focus-visible (quando navighi da tastiera). Questo permette interazioni senza JavaScript, migliorando anche l’accessibilità.

Infine, le variabili CSS (es. --accent) rendono lo stile più “pulito”: cambi un colore una volta sola e si aggiorna ovunque. Qui le variabili sono dichiarate in :root.

Nota: tutto il CSS di questa pagina è dentro il tag <style>. In un progetto reale spesso lo metti in un file separato (es. styles.css) e lo colleghi con <link rel="stylesheet">.
